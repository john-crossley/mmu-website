Overview - Accessibility and Usability

3rd October, 2012 by John Crossley

When developing websites it’s important everyone should have equal access this goes for people with disabilities such as partial blindness, blindness, colour blindness etc. So this was an important factor to consider whilst building this website site (After all it’s a legal requirement anyways) In order to achieve this I built in several different methods to help aid those who are less able to use the website to it’s full potential.

Aria Roles

These allow a screen reader to pickup certain elements of the page and allow the user to navigate to these to read the content. Because screen readers read in a linear fashion aria roles allow the user to navigate to different sections of the page making it quicker for a user to navigate to the content he/she wants.

Skip Links

To make navigation even quicker I have implemented skip links, this allows the user to quickly navigate to sections of the site as soon as the page has loaded. This way he/she can find the content they want immediately

Testing

My website has been tested on a number of devices including iPad, iPhone / iPhone 4S with Retina display, iMac 21.5inch with 1920x1080 resolution and a computer with the resolution of 1440x900 and the site responsively fits each device. and for accessibility I have tested the website using a free online screen reader called http://webanywhere.cs.washington.edu/wa.php this allowed me to test and tweak my website to adhear the disability relating standards
